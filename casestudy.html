Case Study - My Movies App

Using the Javascript development of the MERN (MongoDB, Express, React, Node.js) stack, I created an application allows users to view information about different movies including the details on the genre and director. They are also able to update their own personal information and can create their own list of favourite movies from the selection given. The goal by the end was to ensure the following features of the app were implemented successfully:

Allow new users to register and log in

Return a list of movies to all users

Return data (description, genre, director, image URL) about a single movie by title to the user

Return data about a genre (description) by name/title

Allow users to add and delete movies from their list of favourites

Return data about a director (bio, birth year, death year) by name

Allow users to update their user info or delete their account



WHAT WENT WELL:

My strengths in this project occurred with building the foundation of the app. I was successful in storing and retrieving data and creating separate components and pages to ensure individual focus on each function including user registration and log in, movie and profile viewing, and updating profiles. Presentation and documentation of coding is important when having to navigate in between scripts and finding bugs.

Skills

  • Organized all coding to make it easily readable, with notes provided to track the functions of each code
  • Consistency in collecting movie database and updating where necessary

Processes

  • Created a REST API using the MERN (MongoDB, Express, React, Node.js
  • Using CRUD (Create, Read, Update, Delete) to retrieve data from my database and store that data in a non-relational way

Solutions

  • Remain consistent with each technique

WHAT DIDN'T GO WELL:

The greatest challenge in this project came down to testing the business logic layer of the app – ensuring that it could recognize a registered user and allow the user to log in and make updates. My statements and queries were not being read and as a result, the data appeared null. This obstacle reappeared when it came to client-side app routing. Some of my functions were not properly defined, making it difficult to call data between the components, API, and the front end.

Skills

  • Translating between Node.js application and MongoDB database using Mongoose
  • State-based routing

Processes

  • Implementing authentication and authorization

Solutions

  • Review reading sources from exercise
  • Reach out to tutor, mentor, and study buddies for assistance

WHAT CAN BE IMPROVED:

I need to ensure that all my data, functions, endpoints, and routes consistently align between the front, middle, and back end of the app. Even the smallest of errors such as inconsistent capitalization on a property can prevent the app from being displayed properly. Having a consistent workflow and visual strategic map will help me resolve bugs more efficiently.

Skills

  • Defining API endpoints and ensuring they match on both the front and back end

Processes

  • Utilize the workflow of creating a component, making an API, calling the display data, and repeat

Skills Gap

  • Project and time management – need to outline what areas of the project need to be prioritized and finding the most efficient building sequence

Solutions

  • Create new, similar database apps but different subjects (i.e. music, travel) to get more comfortable with both server-side and client-side app building

CAREER CHANGE STRATEGY:

  • Revisit the app frequently in order to make enhancements to maintain and improve my skills in full stack Javascript development
  • Build new API databases for future, similar projects in order to get comfortable with the MERN process
  • Continue education of full stack web development through online readings and tutorials
  • Network and collaborate with developers at a similar level to work jointly on projects for practice and to support one another in growing our skills in programming
